## State Contracts

### Loading state (D-16)

Render skeleton sections immediately — no blank page, no centered spinner.

```
Tile row: 4× <Skeleton className="h-[80px] rounded-lg" />
Agent table: <Skeleton className="h-[160px]" /> (covers 3 rows)
System facts: 5× <Skeleton className="h-4 w-full" /> with gap-3
Privacy panel: 4× <Skeleton className="h-4 w-full" /> with gap-3
```

### Error state (D-09)

Keep page shell and header visible. Replace content area body with a compact retry panel:

```
<Card className="p-6 flex flex-col items-center gap-3 text-center">
  <p className="text-sm text-foreground">Failed to load diagnostics</p>
  <p className="text-xs text-muted-foreground">Could not reach /api/diagnostics</p>
  <Button variant="outline" size="sm" onClick={reload}>Retry Load</Button>
</Card>
```

No Alert component for the error state — Card + inline copy is sufficient and less alarmist.

### Healthy state

All tiles green dot or neutral values. Agent table shows "configured" badges in green. No warning section visible. Summary tiles show `0 warnings` in muted-foreground.

### Warning state (D-05, D-06)

Tiles with warning counts > 0 display count in foreground color (not red — red is reserved for critical). Individual warnings surface inline in the agent table rows and privacy panel as `Badge` elements.

Warning badge color mapping:
- `degraded` → destructive red
- `missing` (when hook was previously configured) → destructive red
- `unknown` → amber
- `stale` → amber
- `no events` → muted text only, no badge color

### Empty / first-run state (D-11)

When `totalEvents === 0` AND all agents show `no events`:

```
<div className="flex flex-col items-center gap-2 py-6 text-center">
  <p className="text-sm text-foreground">No activity observed yet</p>
  <p className="text-xs text-muted-foreground">
    Run <code className="font-mono">hooker setup</code> or <code className="font-mono">hooker doctor</code> to configure hook integrations.
  </p>
</div>
```

Shown inside the agent table section. Summary tiles still render with zero counts.

### Not-ready state (D-10)

Render all available diagnostics normally. Readiness tile shows red dot + "Not ready" label + `reason` string in 12px muted text below. Do not hide other sections.

### Refresh state (D-13, D-14, D-15)

- `refreshing: true` → RefreshCw icon has `animate-spin` class, Button has `disabled` prop
- Current data remains visible during refresh — no skeleton re-render
- "Updated {relative time}" label updates after successful reload
- Timestamp format: "Updated just now" / "Updated 2m ago" — using `formatDistanceToNow` from `date-fns`

## Privacy / Security Panel Contract

Two stacked Card sections inside the right column.

### System Facts card

Fact rows (label: value format, Separator between rows):
- Version: `version.version` + commit hash (monospace, 8 chars) + build date
- DB Path: monospace, truncated with `title` attribute for full path + copy icon
- DB Size: formatted bytes (e.g. "12.4 MB") or reason string if unavailable
- Total Events: integer formatted
- Total Sessions: integer formatted
- Latest Event: relative timestamp or "None"

### Privacy & Security card (D-17, D-18, D-19)

Fact rows with small status badges inline:
- Ignore file: path (monospace, truncated, copy icon) + status badge
  - `loaded` → green "Active"
  - `missing_ok` → muted "Not configured" (no warning — D-12)
  - `missing` → amber "Missing"
  - `error` → red "Error"
- Active rules: `activePatternCount` as integer
- Bind posture: addr string + status badge
  - `loopback` → green "Loopback only"
  - `remote` → red "Remote enabled" (D-19)
- CORS origins: `totalOrigins` total, `localOrigins` local, `extraOrigins` extra
  - When `extraOrigins > 0` → amber badge "N extra origins" (D-19)
  - When `extraOrigins === 0` → green badge "Local only"

Export sensitivity warning (D-18) — always visible, below privacy facts:
```
<Alert className="mt-4 border-[var(--border)] bg-[var(--secondary)]">
  <AlertDescription className="text-xs text-muted-foreground">
    {privacy.exportWarning}
  </AlertDescription>
</Alert>
```

No warning icon on this Alert — it is informational, not alarming. Use the backend-provided `exportWarning` string verbatim.

## Monospace Path Contract (D-20)

All file paths (DB path, ignore file path) use:

```tsx
<span
  className="font-mono text-[12px] text-foreground truncate max-w-[220px] inline-block align-bottom"
  title={fullPath}
>
  {fullPath}
</span>
<button
  onClick={() => navigator.clipboard.writeText(fullPath)}
  className="ml-1 opacity-40 hover:opacity-100 transition-opacity"
  aria-label="Copy path"
>
  <Copy className="size-3 inline" />
</button>
```

The `title` attribute provides full-value access without a tooltip component (avoids tooltip nesting complexity). The copy button is a plain `<button>` — not a shadcn Button — to keep the inline affordance minimal.

## Hook Contract

New file: `frontend/src/features/diagnostics/hooks/useDiagnostics.ts`

Shape mirrors `useDashboardStats` exactly:

```ts
export function useDiagnostics(): {
  data: Diagnostics | null
  loading: boolean
  refreshing: boolean
  lastUpdatedAt: Date | null
  reload: () => void
  error: string | null
}
```

- `loading`: true only on first fetch (no cached data)
- `refreshing`: true during subsequent fetches while data exists
- `error`: string message when fetch fails, null otherwise
- `lastUpdatedAt`: Date of last successful response (drives "Updated ..." display)
- `reload`: increment reloadKey to trigger re-fetch

No caching — diagnostics is a live snapshot. Fetch on mount and on manual reload only (D-13).

**Source:** `useDashboardStats` pattern from `frontend/src/features/dashboard/hooks/useDashboardStats.ts`.

## Warning Count Computation

Tile 3 (Agent warnings) counts: agents where `status === 'degraded'` OR (`hookConfigStatus === 'missing'`) OR (`hookConfigStatus === 'unknown'` AND `eventCount === 0`).

Tile 4 (Privacy warnings) counts: `security.remoteBind.allowRemote === true` (1) + (`security.cors.extraOrigins > 0` ? 1 : 0) + (`privacy.ignoreFile.status === 'error'` ? 1 : 0).

"No events" agents (all agents with `status === 'no events'`) are never counted as warnings (D-07). `hookConfigStatus === 'unknown'` alone is not counted as a warning (D-08) unless paired with `eventCount === 0`.

## Accessibility Contract

- Page heading `<h1>` present on every render state (loading/error/loaded)
- Skeleton sections have `aria-busy="true"` on their container
- Refresh button `aria-label="Refresh diagnostics"`, `disabled` during refresh
- Retry button keyboard-focusable, not icon-only
- Agent table uses semantic `<table>` with `<th scope="col">` headers
- All badge-only status indicators have tooltip or `title` fallback with full status text
- Monospace paths: `title` attribute on truncated span (no tooltip dependency)
- Copy buttons: `aria-label="Copy {field name}"` e.g. `aria-label="Copy DB path"`
- Color is never the sole differentiator — badges always include text label
