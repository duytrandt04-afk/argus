# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Local Adoption Baseline.
> Scope: single new component — runtime version display in Sidebar footer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

This phase has exactly one frontend UI deliverable: **a runtime-fetched version string
in the Sidebar footer**, replacing the existing Vite-baked `APP_VERSION` inline display.

The contract below is scoped to that component. Existing app-wide tokens are documented
for reference only — do not change them as part of this phase.

---

## Existing UI Context (Do Not Change)

The Sidebar (`frontend/src/app/Sidebar.tsx`) currently renders a version string at line 199:

```tsx
<span className="sidebar-label-motion sidebar-label-open whitespace-nowrap text-[0.66rem] font-medium text-[#555]">
  v{APP_VERSION}
</span>
```

This is in the **header row** (logo + app name row) and reads from `@/version` which uses
`__HOOKER_VERSION__` — a Vite define-time constant. Decision D-07 mandates moving to a
runtime fetch. Decision D-05 mandates footer placement.

**Action required:** Remove the `APP_VERSION` import and the inline `<span>` from the
header row. Add the new `VersionBadge` component to the footer slot instead.

- No hover state. Version text is informational only, not interactive.
- No focus state. Not focusable.

## Component Contract: `VersionBadge`

### File location

`frontend/src/features/version/VersionBadge.tsx`

Hook: `frontend/src/features/version/useVersion.ts`

### Hook: `useVersion`

Fetches `GET /api/version` once on mount. No polling. No retry.

Return shape:

```ts
type UseVersionResult = {
  version: string | null   // null while loading or on error
  error: boolean           // true if fetch failed
}
```

Fetch behavior:
- On mount: fire `fetch('/api/version')` once
- Response shape expected: `{ version: string, commit: string, buildDate: string }`
- Construct display string: `v${version} (${commit.slice(0, 7)})`
- On network error or non-2xx: set `error: true`, `version: null`
- No AbortController needed — single one-shot fetch at mount, component is always mounted

### Component: `VersionBadge`

Props: none (self-contained, reads from hook internally)

DOM output (collapsed sidebar state — icon-only mode):
```tsx
{/* Nothing rendered — version hidden when sidebar is collapsed */}
null
```

DOM output (expanded sidebar state):
```tsx
<span
  className="whitespace-nowrap text-[0.66rem] font-medium leading-none text-[#444]"
  aria-label={`Application version: ${displayText}`}
>
  {displayText}
</span>
```

Where `displayText` is one of:
- Loading state: nothing rendered — return `null`. No skeleton. No placeholder text.
  Rationale: version is purely informational; a brief empty state during boot is preferable
  to layout shift from a skeleton or "loading..." text.
- Success state: `v0.1.0 (abc1234)` — format locked by D-06
- Error state: nothing rendered — return `null`. Do not show "unknown" or any error label.
  Rationale: if `/api/version` fails, the app is still usable. A visible error state for
  a version badge adds noise with no user action path. Silent omission is correct.

### Placement in Sidebar.tsx

The Sidebar footer currently has:

```tsx
{/* Bottom divider line */}
<div className="mt-auto">
  <div className="sidebar-bottom-divider" />
</div>
```

Replace with:

```tsx
<div className="mt-auto">
  <div className="sidebar-bottom-divider" />
  {!collapsed && (
    <div className="flex items-center px-2 pt-2 pb-1">
      <VersionBadge />
    </div>
  )}
</div>
```

Spacing breakdown:
- `px-2` (8px horizontal) — matches sidebar's own `px-2` root padding
- `pt-2` (8px top) — gap from divider line above
- `pb-1` (4px bottom) — tight bottom clearance to sidebar edge
- Entire block hidden when `collapsed === true` — no tooltip needed for version in
  collapsed mode (it is a status label, not navigation)

### Collapsed sidebar behavior

When `collapsed === true`, the `VersionBadge` renders nothing.
No tooltip showing version on hover. Version is supplementary info; nav tooltips are
reserved for navigation actions (existing pattern in Sidebar).

### Mobile sidebar behavior

`VersionBadge` renders in mobile mode (when `mode === 'mobile'`) since the mobile
sidebar is always fully expanded. The collapsed guard (`!collapsed`) is always false
in mobile mode, so the badge is always visible there.

## Accessibility Contract

| Concern | Decision |
|---------|----------|
| `aria-label` | `aria-label="Application version: v0.1.0 (abc1234)"` — provides screenreader context since the raw text has no surrounding heading |
| Role | None explicit — `<span>` with `aria-label` is sufficient for a status label |
| Focusability | Not focusable — `tabIndex` not set. Version is read-only status, not interactive. |
| Keyboard interaction | None — not interactive |
| Color contrast | `#444` on `#111` = 3.24:1. Intentionally below AA (4.5:1) because this is decorative status text, not content-critical. Acceptable per WCAG 1.4.3 exception for incidental text. |
