# Milestone v1.1: Diagnostics

**Status:** ✅ SHIPPED 2026-05-29
**Phases:** 4–6
**Total Plans:** 9

## Overview

v1.1 adds an operator-focused Diagnostics feature to hooker: a read-only backend contract at `GET /api/diagnostics` reporting system health, version metadata, storage facts, aggregate event/session counts, Claude Code and Codex agent telemetry, hook config status, privacy ignore posture, remote-bind/CORS posture, and export sensitivity warning — all surfaced in a compact React Diagnostics page reachable from the sidebar.

## Phases

### Phase 4: Diagnostics Data Contract

**Goal**: Add a read-only backend diagnostics contract that reports system health, version metadata, storage facts, and aggregate event/session stats without expensive full-table scans.
**Depends on**: v1.0 MVP
**Requirements**: DIAG-01, DIAG-02, DIAG-03, DIAG-04, TEST-01
**Plans**: 3 plans

Plans:

- [x] 04-01: Domain response shape + repository diagnostics aggregate queries
- [x] 04-02: Service/handler wiring + `/api/diagnostics` route
- [x] 04-03: Backend tests for system health, storage stats, empty DB, and response shape

**Details:**

- Typed diagnostics response structs with grouped `version`, `health`, and `storage` sections.
- `EventRepository.DiagnosticsStorageStats()` for diagnostics-specific aggregate reads.
- SQLite aggregate queries using `COUNT(*)` and `MAX(created_at)` — no full-table scans.
- Diagnostics inspectable on readiness failure (HTTP 200, `health.ready: false`).
- Handler tests cover grouped JSON shape, not-ready behavior, aggregate error, and captured-content non-leakage.

## Milestone Summary

**Resolved at Audit:**

- HOOK-01: Gemini CLI removed from requirement scope per user decision. Backend emits only Claude Code and Codex rows; frontend test fixtures aligned.
- Status mismatch: `event_service.go` `Status: "ok"` corrected to `Status: "healthy"` so `AgentStatusCell` renders the correct badge.

**Key Decisions:**

- Diagnostics storage aggregates use a dedicated `DiagnosticsStorageStats()` repository method, not the dashboard or list flows.
- Nullable values use pointer fields (`*int64`, `*string`) so JSON encodes `null` for unavailable size and empty latest event.
- Diagnostics remains inspectable on readiness failure (HTTP 200 with `health.ready: false`).
- `hookconfig` package is a separate Go package (not a shell-out to `scripts/hooker`) for testability and reuse.
- `LoadedContent` sub-component extracts TypeScript narrowing so `data` is non-null throughout JSX.
- `loading-only` condition for skeleton (not `loading || refreshing`); refreshing only spins the button icon.
- Gemini CLI diagnostics deferred from HOOK-01 scope; requirement updated before milestone close.

**Issues Resolved:**

- Blank page crash when agent has no warnings: `DiagnosticsAgent.Warnings` slice initialized to empty array instead of nil in Go.
- React StrictMode phantom effect causing state mutation race in `useDiagnostics` mounted guard in finally block.
- 9 code review findings applied to DiagnosticsPage and Sidebar (NavButton extraction, formatBytes validation, Button component usage, warning count calculation).

**Tech Debt Incurred:**

- Phase 6 SUMMARY.md files missing `requirements-completed` frontmatter field (covered by VERIFICATION.md).
- No VALIDATION.md (Nyquist) generated for phases 4, 5, or 6.
